# Как применить ФОС за 30 минут

## 1. Подготовка

1. Создайте новый репозиторий на основе заполненного шаблона.
2. Укажите официальный код дисциплины в имени `fos-<код-дисциплины>`.
3. Проверьте сведения в `meta/inputs.md`, `meta/roles_scope.md` и `docs/passport.md`.
4. Проверьте суммы весов по `tests/weights_check.md`.
5. Проверьте покрытие индикаторов по `tests/coverage_check.md` и
   `data/competency_map/mapping.csv`.

## 2. Настройка учебной группы

1. Зафиксируйте календарь 12 лабораторных работ Р1.1-Р6.2.
2. Назначьте оценочные средства и ответственных проверяющих.
3. Для peer-review используйте `docs/peer_review.md` и
   `src/templates/peer_review_form.md`.
4. Для итогового проекта используйте `docs/project_criteria.md`,
   `src/rubrics/project_ps.md` и `src/rubrics/project_doc.md`.
5. Для защиты используйте `docs/commission_criteria.md` и
   `src/templates/commission_form.md`.

## 3. Расчёт результатов

Лабораторный блок:

`ОС-1 = Σ(hᵢ × bᵢ) / 28`.

Итоговая оценка:

`Итог = 0.20×ОС1 + 0.10×ОС2 + 0.10×ОС3 + 0.15×ОС4 + 0.30×ОС5 + 0.15×ОС6`.

Обязательные пороги:

- итог не ниже 4,0;
- программное решение итогового проекта не ниже 4;
- документация итогового проекта не ниже 4.

## 4. Перед защитой

- [ ] репозитории студентов доступны комиссии;
- [ ] все обязательные артефакты проекта присутствуют;
- [ ] суммы весов равны 100%;
- [ ] каждый индикатор оценивается не менее чем двумя средствами;
- [ ] подготовлены печатные формы из `release/printable/`;
- [ ] назначены члены комиссии и установлен регламент;
- [ ] подготовлена резервная демонстрация на случай аппаратного сбоя.

## 5. После семестра

1. Зафиксируйте замечания через `.github/ISSUE_TEMPLATE/methodological_issue.md`.
2. Обновите `validation/report.md` по фактическим результатам.
3. Внесите изменения в `CHANGELOG.md`.
4. Существенные методические решения оформляйте ADR в `meta/decisions/`.
