# Фонд оценочных средств дисциплины
## «Инженерия встроенных систем машинного обучения»

Репозиторий содержит заполненный фонд оценочных средств (ФОС) проектной работы
дисциплины. Структура повторяет структуру шаблона ФОС: постановка задачи,
метаданные, паспорт, матрица компетенций, рубрики, заполняемые формы, проверки,
пилотная валидация и комплект релиза.

## Основные сведения

| Параметр | Значение |
|---|---|
| Уровень образования | бакалавриат |
| Направление подготовки | 09.03.02 / 09.03.01 |
| Профиль | Инженерия промышленных систем искусственного интеллекта |
| Курс / семестр | 4 / 7 |
| Трудоёмкость | 3 ЗЕ, 108 академических часов |
| Профессиональная роль КРМ | ML Engineer |
| Индикаторы | `PL-4.2`, `DL-1.11`, `LC-5.1`, `DL-3.3` |
| Отраслевые траектории | ТЭК, Транспорт |
| Лабораторный блок | 12 работ, 28 академических часов по РПД |
| Шкала | 0-10; опорные уровни 4 / 6 / 8 / 10 |

## Быстрый старт

1. Откройте [FOS_BRIEF.md](FOS_BRIEF.md) и [meta/inputs.md](meta/inputs.md).
2. Проверьте паспорт в [docs/passport.md](docs/passport.md).
3. Используйте матрицу в [docs/matrix.md](docs/matrix.md) и машиночитаемый
   вариант [data/competency_map/mapping.csv](data/competency_map/mapping.csv).
4. Для итогового проекта используйте [docs/project_criteria.md](docs/project_criteria.md).
5. Для взаимного рецензирования и защиты используйте формы из
   [src/templates/](src/templates/).
6. Перед применением выполните ручные проверки из [tests/](tests/).
7. Инструкция преподавателю находится в [release/how_to_apply.md](release/how_to_apply.md).

## Лабораторные работы и расчёт

В РПД предусмотрены 12 лабораторных работ общей трудоёмкостью 28 часов:

| Работа | Часы | Индикатор | Краткая тема |
|---|---:|---|---|
| Р1.1 | 3 | PL-4.2 | ESP32-S3, PlatformIO/ESP-IDF, сенсоры и GPIO |
| Р1.2 | 3 | PL-4.2 | питание, DMA, прерывания и STM32Cube.AI |
| Р2.1 | 2 | DL-3.3 | калибровка датчиков и управление актуаторами |
| Р2.2 | 2 | DL-3.3 | BLE/Wi-Fi и предобработка телеметрии |
| Р3.1 | 2 | LC-5.1 | FreeRTOS, приоритеты задач и ML-инференс |
| Р3.2 | 2 | LC-5.1 | OTA и логирование метрик инференса |
| Р4.1 | 3 | DL-1.11 | INT8-квантование и калибровочная выборка |
| Р4.2 | 3 | DL-1.11 | автоматизация сжатия и профилирование |
| Р5.1 | 2 | DL-3.3 | Edge Impulse / TFLite Micro |
| Р5.2 | 2 | DL-3.3 | Object Detection и RKNN-Toolkit2 |
| Р6.1 | 2 | LC-5.1, PL-4.2 | ток, latency и энергосбережение |
| Р6.2 | 2 | LC-5.1, PL-4.2 | стабильность инференса при сбоях питания |
| **Итого** | **28** |  |  |

Лабораторный блок составляет 20% итоговой оценки. Балл блока:

`ОС-1 = Σ(hᵢ × bᵢ) / 28`,

где `hᵢ` - часы работы по РПД, `bᵢ` - оценка работы по шкале 0-10.
Подробная привязка лабораторных работ к компетенциям и оценочным средствам
включена в `docs/matrix.md` и `data/competency_map/mapping.csv`.

## Нижние пороги

Дисциплина считается освоенной при итоговом балле не ниже 4,0. По каждой оси
итогового проекта - «Программное решение» и «Документация» - требуется не менее
4 баллов.
